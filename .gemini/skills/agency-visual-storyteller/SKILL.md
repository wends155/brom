# Visual Storyteller

## When to Use
Load this skill during the Intent & Discovery phase of the `/design` workflow (Step 1) when GUI or Assets mode is active, to establish narrative flow and information architecture.

## Information Hierarchy Framework
- **Primary Information**: Critical data storytelling, essential for immediate task completion or core message.
- **Secondary Information**: Supporting charts/graphs and details that provide context.
- **Tertiary Information**: Background data and deep-dive analytics for power users.

## Progressive Disclosure Patterns
*(Composed from UX design principles)*
- **Accordion**: Collapsing secondary details to keep the primary view scannable.
- **Drill-down**: Navigation from summary views into detailed data views.
- **Staged Forms**: Breaking complex input flows into digestible, logical steps.
- **Contextual Help**: Providing guidance only when a user interacts with a feature.
- **Tooltip Reveals**: Hiding definitions or minor context behind hover states.

## Narrative Arc for UI
- **Setup (Landing/Entry)**: The protagonist (user) arrives with a specific goal, encountering the brand/product interface. Map to: landing page, onboarding flow, or dashboard home. Key question: "What is the user's first impression and immediate orientation?"
- **Tension (Task Complexity)**: The problem or challenge the user must navigate through the UI flow. Map to: multi-step forms, search/filter workflows, or configuration screens. Key question: "Where does cognitive load peak, and how do we reduce friction?"
- **Resolution (Completion)**: The success state where the product provides the solution and emotional payoff. Map to: confirmation screens, result displays, or summary views. Key question: "Does the user feel accomplished and know what to do next?"

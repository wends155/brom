# Whimsy Injector

## When to Use
Load this skill during the Polish & Delight phase of the `/design` workflow (Step 3) when GUI mode is active, to add personality and micro-interactions.

## Micro-Interaction Taxonomy
- **Subtle Whimsy**: Small touches that add personality without distraction (e.g., Hover effects, loading animations, button feedback)
- **Interactive Whimsy**: User-triggered delightful interactions (e.g., Click animations, form validation celebrations, progress rewards)
- **Discovery Whimsy**: Hidden elements for user exploration (e.g., Easter eggs, keyboard shortcuts, secret features)
- **Contextual Whimsy**: Situation-appropriate humor and playfulness (e.g., 404 pages, empty states, seasonal theming)

## Interaction State Framework
Ensure UI state transitions feel alive and responsive:
1.  **Entrance/Exit**: Don't just pop elements in. Use cubic-bezier easing to slide/fade.
2.  **Success/Completion**: Add particle effects, bounces, or color bloom on critical path completions (e.g., submitting a form, checking a task).
3.  **Loading/Waiting**: Use skeleton loaders with shimmering gradients instead of static spinners, or brand-specific loading animations.
4.  **Error/Failure**: Provide gentle, non-punitive "blips" or soft shakes.

## Playful Microcopy Patterns

### Error Messages
- **404 Page**: "Oops! This page went on vacation without telling us. Let's get you back on track!"
- **Form Validation**: "Your email looks a bit shy – mind adding the @ symbol?"

### Empty States
- **No Search Results**: "No matches found, but your search skills are impeccable!"
- **Empty Cart**: "Your cart is feeling a bit lonely. Want to add something nice?"

### Success Messages
- **Form Submission**: "High five! Your message is on its way."
- **Task Completion**: "Boom! You're officially awesome."

### Loading States
- **General Loading**: "Sprinkling some digital magic..."
- **Data Processing**: "Crunching numbers with extra enthusiasm..."
